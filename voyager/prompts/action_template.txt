===============================
RULES FOR PROGRAM GENERATION
===============================

1. You MUST output exactly ONE complete async function.
   - No helper functions.
   - No gathering sub-skills.
   - No “mineWoodLog()”, “gatherStone()”, or ANY wrapper functions.

OR You may output exec_code instead of defining a new async function.
Use exec_code when the task requires only a direct primitive action.
Example:
{{
  "program_code": "",
  "program_name": "",
  "exec_code": "await mineBlock(bot, 'stone', 3);",
  "reasoning": "...",
  "recipe": null
}}


2. You MUST call Mineflayer primitives DIRECTLY:
   - await mineBlock(bot, '<block_name>', <count>);
   - await craftItem(bot, '<item_name>', <count>);
   - await smeltItem(bot, '<item_name>', <count>);
   - await placeItem(bot, '<item_name>', <Vector3>);
   - await killMob(bot, '<mob_name>', <count>);
   - etc.

3. NEVER create new functions ONLY to gather resources.
   - Gathering must ALWAYS be done using direct primitive calls in the program_code function.

4. DO NOT check inventory.
   DO NOT test for missing materials.
   DO NOT add flow-control to manage materials.
   - Just call primitives directly.
   - The system (HTN) handles missing prerequisites and retries.

5. You MAY call known skills ONLY if they perform multi-step transformations.
   - e.g., “smeltIronIngot” is allowed.
   - You may NOT create your own gathering or mining skills.

6. All primitive calls MUST include full arguments and MUST be valid calls:
   - mineBlock(bot, 'stone', 3)
   - craftItem(bot, 'oak_planks', 4)

7. Your async function must use bot.chat() to announce actions.
